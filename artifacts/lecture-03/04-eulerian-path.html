Eulerian Path Finder

Check whether a directed graph has an Eulerian path or circuit, then watch Hierholzer's algorithm walk it step by step. Use a preset, or paste your own edges. The analyzer names the exact reason a graph fails — a node with imbalance > 1, too many unbalanced nodes, or disconnection.

Graph source

Nodes are any strings (ACG, A, node42). Multiple edges between the same nodes are fine. If a k-mer isn't supplied, the edge label is the FROM node concatenated with the last character of the TO node.

Walk

Hierholzer's walks greedily until stuck, then backtracks through the stack to find a node with unvisited edges and splices a new subpath in. Edges used are shown in cobalt; unvisited edges in gray. Dashed node outline marks the detected start (cobalt) or end (amber) of an Eulerian path.
