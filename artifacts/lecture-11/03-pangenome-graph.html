Pangenome Graph Viewer

A pangenome represents multiple haplotypes as a single branching graph. Shared sequence collapses into one node; variation becomes parallel branches. Toggle between the linear view (each haplotype as its own string) and the graph view (a unified structure with paths traced through it) to see why a graph is a more compact and honest representation of variation than any single reference.

Pangenome view

Linear view · each haplotype as a separate string

N haplotypes stacked vertically. Bases coloured A/C/G/T. Variant sites highlighted with vertical bands. Notice how much sequence is identical across rows — this redundancy is what the graph view will collapse.

Graph view · shared sequence collapses, variants branch

Backbone nodes (grey) are shared by all haplotypes. Variant sites split into parallel branching nodes, one per allele. Each haplotype is a coloured path that picks one branch at every variant site.

Variant table · which haplotypes carry which alleles

Each row is one variant site. The alleles column lists the distinct bases (or presence/absence for indels) observed. The per-haplotype columns show which allele each haplotype carries at that site. The graph view is this table made geometric.
